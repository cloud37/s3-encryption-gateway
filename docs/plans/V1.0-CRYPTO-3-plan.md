## 2. Detailed Design

### 2.1 Key Format and Loading

The metadata encryption key is a **256-bit (32-byte) AES key**. It can be provided in exactly one of three mutually exclusive ways:

#### Option A: File (preferred for non-KMS deployments)

```yaml
encryption:
  metadata_encryption_key_file: "/etc/s3eg/metadata-master.key"
```

File contents: a single base64-encoded 32-byte random value:

```bash
openssl rand -base64 32 > /etc/s3eg/metadata-master.key
chmod 600 /etc/s3eg/metadata-master.key
```

Validation at startup:
- File must be readable
- Base64 decode must produce exactly 32 bytes
- Log a warning (not error) if file mode is more permissive than `0600`

#### Option B: Inline (for Helm / Vault / sealed-secret deployments)

```yaml
encryption:
  metadata_encryption_key: "gKf3zB0q..."   # ≥ 128 ASCII characters
```

Validation at startup:
- Minimum 128 ASCII characters
- SHA-256 hash the string to produce exactly 32 bytes
- The 128-char minimum is a **policy guardrail** against low-entropy passwords; cryptographically the bottleneck is always the 256-bit AES key

**Why inline is supported despite the file preference:**
In Kubernetes/Helm environments, `valueFrom` and external secret managers (Vault,
HelmSecrets, Sealed Secrets) often inject sensitive values directly into Pod env
vars or ConfigMaps. Requiring a file mount adds operational friction in these
environments. The inline option preserves deployment flexibility while keeping
strict validation.

#### Option C: KMS wrapping (when `key_manager` is enabled)

If a `KeyManager` is configured, the gateway derives or loads the metadata key,
wraps it via `KeyManager.WrapKey`, and stores the envelope in the engine. On
startup the envelope is unwrapped via `KeyManager.UnwrapKey`. The plaintext key
never leaves gateway memory. This is the recommended option for production
because it inherits KMS features: HA, rotation, audit logging.

**Mutual exclusion:** exactly one of `metadata_encryption_key_file`,
`metadata_encryption_key`, or KMS wrapping may be active. Startup fails if two
or more are set.

### 2.2 Metadata Encryption Format (`internal/crypto/metadata_encrypt.go`)

#### Data Flow (Write)

```
encMetadata map (all keys)
    │
    ▼
Extract encryption + compression keys
    (IsEncryptionMetadata || IsCompressionMetadata)
    │
    ▼
{ "encrypted": true, "algorithm": "...", "salt": "...", ... }
    │
    ▼
AES-256-GCM Seal(nonce=random-12B, key=metadataKey, plaintext=json)
    │
    ▼
base64(nonce || ciphertext || tag)
    │
    ▼
encMetadata["x-amz-meta-enc-metadata"] = blob
(encMetadata no longer contains individual encryption keys)
```

#### Data Flow (Read)

```
metadata map from S3
    │
    ▼
Check for "x-amz-meta-enc-metadata" (or compacted "x-amz-meta-em")
    │
    ▼
base64-decode → nonce(12B) || ciphertext || tag(16B)
    │
    ▼
AES-256-GCM Open(key=metadataKey, nonce, ciphertext, tag)
    │
    ▼
JSON parse → map[string]string
    │
    ▼
Merge into metadata map
    │
    ▼
Continue with existing decryption logic
```

#### Code Sketch

```go
package crypto

import (
    "crypto/aes"
    "crypto/cipher"
    "crypto/rand"
    "encoding/base64"
    "encoding/json"
    "fmt"
    "io"
)

const (
    MetaEncryptedMetadata = "x-amz-meta-enc-metadata"
    MetaEncryptedMetadataCompact = "x-amz-meta-em"
)

func (e *engine) encryptMetadata(encMeta map[string]string) (string, error) {
    if e.metadataKey == nil {
        return "", fmt.Errorf("metadata key not configured")
    }

    // Extract encryption and compression metadata
    subset := make(map[string]string)
    for k, v := range encMeta {
        if IsEncryptionMetadata(k) || IsCompressionMetadata(k) {
            subset[k] = v
        }
    }

    // Serialize
    plaintext, err := json.Marshal(subset)
    if err != nil {
        return "", fmt.Errorf("marshal metadata: %w", err)
    }

    // Seal
    block, err := aes.NewCipher(e.metadataKey)
    if err != nil {
        return "", fmt.Errorf("aes new cipher: %w", err)
    }
    gcm, err := cipher.NewGCM(block)
    if err != nil {
        return "", fmt.Errorf("gcm: %w", err)
    }

    nonce := make([]byte, gcm.NonceSize())
    if _, err := io.ReadFull(rand.Reader, nonce); err != nil {
        return "", fmt.Errorf("nonce: %w", err)
    }

    ciphertext := gcm.Seal(nonce, nonce, plaintext, nil)
    return base64.StdEncoding.EncodeToString(ciphertext), nil
}

func (e *engine) decryptMetadata(blob string) (map[string]string, error) {
    if e.metadataKey == nil {
        return nil, fmt.Errorf("metadata key not configured")
    }

    ciphertext, err := base64.StdEncoding.DecodeString(blob)
    if err != nil {
        return nil, fmt.Errorf("base64 decode: %w", err)
    }

    block, err := aes.NewCipher(e.metadataKey)
    if err != nil {
        return nil, fmt.Errorf("aes new cipher: %w", err)
    }
    gcm, err := cipher.NewGCM(block)
    if err != nil {
        return nil, fmt.Errorf("gcm: %w", err)
    }

    nonceSize := gcm.NonceSize()
    if len(ciphertext) < nonceSize {
        return nil, fmt.Errorf("ciphertext too short")
    }

    nonce, ciphertext := ciphertext[:nonceSize], ciphertext[nonceSize:]
    plaintext, err := gcm.Open(nil, nonce, ciphertext, nil)
    if err != nil {
        return nil, fmt.Errorf("gcm open: %w", err)
    }

    var result map[string]string
    if err := json.Unmarshal(plaintext, &result); err != nil {
        return nil, fmt.Errorf("unmarshal: %w", err)
    }
    return result, nil
}
```

**JSON schema of encrypted payload:**

The JSON object stores the full key names (not compacted) so that after
 decryption the map can be used directly without a separate expansion step.

```json
{
  "x-amz-meta-encryption-algorithm": "AES256-GCM",
  "x-amz-meta-encryption-key-salt": "+/jFvuPKmb/c/VSVJGQaODpmVDnbkN7coCxHprNmVKY=",
  "x-amz-meta-encryption-iv": "IIX38ktaSaSGzABb",
  "x-amz-meta-encryption-original-size": "14593",
  "x-amz-meta-encryption-kdf-params": "pbkdf2-sha256:600000",
  "x-amz-meta-encryption-chunked": "true",
  "x-amz-meta-encryption-chunk-size": "65536",
  "x-amz-meta-encryption-manifest": "eyJ2Ijox...",
  ...
}
```

### 2.3 Encryption Path Changes

All four encryption entry points are modified identically:

1. `Encrypt` (buffered, line ~325)
2. `encryptChunked` (streaming, line ~781)
3. `encryptWithMetadataFallback` (V1 fallback, line ~1424)
4. `encryptChunkedWithMetadataFallback` (V2 fallback, line ~965)

**Insertion point:** After `encMetadata` is fully populated and before
`e.compactor.CompactMetadata(encMetadata)`.

```go
// If metadata encryption is enabled, encrypt the encryption-internal subset.
if e.metadataKey != nil {
    blob, err := e.encryptMetadata(encMeta)
    if err != nil {
        return nil, nil, fmt.Errorf("encrypt metadata: %w", err)
    }
    // Remove all encryption/compression keys from the cleartext map.
    for k := range encMeta {
        if IsEncryptionMetadata(k) || IsCompressionMetadata(k) {
            delete(encMeta, k)
        }
    }
    encMeta[MetaEncryptedMetadata] = blob
}
```

### 2.4 Decryption Path Changes

All three decryption entry points are modified identically:

1. `Decrypt` (line ~555)
2. `decryptChunked` (line ~1139)
3. `DecryptRange` (line ~1264)

**Insertion point:** Immediately after metadata expansion (compaction reversal),
before `IsEncrypted` check and decryption logic.

```go
// If metadata encryption is enabled, decrypt the encrypted metadata blob.
if e.metadataKey != nil {
    blobKey := MetaEncryptedMetadata
    if _, ok := metadata[MetaEncryptedMetadataCompact]; ok {
        blobKey = MetaEncryptedMetadataCompact
    }
    if blob, ok := metadata[blobKey]; ok && blob != "" {
        decryptedMeta, err := e.decryptMetadata(blob)
        if err != nil {
            return nil, nil, fmt.Errorf("decrypt metadata: %w", err)
        }
        // Merge decrypted metadata back into the main map.
        for k, v := range decryptedMeta {
            metadata[k] = v
        }
        delete(metadata, blobKey)
    }
}
```

### 2.5 Compaction Compatibility

Add `MetaEncryptedMetadata` and `MetaEncryptedMetadataCompact` to the
compaction maps in `internal/crypto/metadata_compaction.go`:

```go
// In compactEncryptionMetadata:
if v, ok := metadata[MetaEncryptedMetadata]; ok {
    compacted[MetaEncryptedMetadataCompact] = v
}

// In expandEncryptionMetadata:
if v, ok := metadata[MetaEncryptedMetadataCompact]; ok {
    expanded[MetaEncryptedMetadata] = v
}
```

This ensures the encrypted metadata blob participates in compaction like any
other encryption metadata key, keeping the total metadata footprint within
provider limits.

### 2.6 IsEncrypted Compatibility

`IsEncrypted` must work even when the gateway does not have the metadata key
(e.g., an old gateway instance reading a new object). The `x-amz-meta-encrypted`
marker remains **outside** the encrypted blob and is never encrypted.

Update `IsEncrypted` to also recognize the compacted alias:

```go
func (e *engine) IsEncrypted(metadata map[string]string) bool {
    if metadata == nil {
        return false
    }
    if encrypted, ok := metadata[MetaEncrypted]; ok && encrypted == "true" {
        return true
    }
    if encrypted, ok := metadata["x-amz-meta-e"]; ok && encrypted == "true" {
        return true
    }
    return false
}
```

No change needed for the compacted alias; it is already covered by the existing
`x-amz-meta-e` check.

### 2.7 Metadata Fallback Path Compatibility

The encrypted metadata blob is just another metadata value. The existing
fallback mechanism handles it transparently:

**V1 fallback (`encryptWithMetadataFallback`):**
- The encrypted blob is part of `encMetadata`.
- `needsMetadataFallback` checks total size including the blob.
- If fallback triggers, the blob is embedded in the object body inside the AEAD
  envelope → attacker needs both the data password and the metadata key.

**V2 fallback (`encryptChunkedWithMetadataFallback`):**
- The encrypted blob is embedded in cleartext before the chunked ciphertext.
- But the blob itself is AES-GCM encrypted → still opaque without the metadata key.

No logic changes to fallback paths.

### 2.8 Startup Wiring (`cmd/server/main.go`)

```go
// After encryptionEngine is created...

var metadataKey []byte

switch {
case cfg.Encryption.MetadataEncryptionKeyFile != "" && cfg.Encryption.MetadataEncryptionKey != "":
    logger.Fatal("Only one of metadata_encryption_key_file or metadata_encryption_key may be set")

case cfg.Encryption.MetadataEncryptionKeyFile != "":
    data, err := os.ReadFile(cfg.Encryption.MetadataEncryptionKeyFile)
    if err != nil {
        logger.WithError(err).Fatal("Failed to read metadata encryption key file")
    }
    key, err := base64.StdEncoding.DecodeString(strings.TrimSpace(string(data)))
    if err != nil {
        logger.WithError(err).Fatal("Metadata encryption key file is not valid base64")
    }
    if len(key) != 32 {
        logger.Fatalf("Metadata encryption key must be exactly 32 bytes (got %d)", len(key))
    }
    metadataKey = key

case cfg.Encryption.MetadataEncryptionKey != "":
    if len(cfg.Encryption.MetadataEncryptionKey) < 128 {
        logger.Fatal("metadata_encryption_key must be at least 128 characters")
    }
    hash := sha256.Sum256([]byte(cfg.Encryption.MetadataEncryptionKey))
    metadataKey = hash[:]
}

if metadataKey != nil {
    if keyManager != nil {
        envelope, err := keyManager.WrapKey(ctx, metadataKey, nil)
        if err != nil {
            logger.WithError(err).Fatal("Failed to wrap metadata key with KMS")
        }
        crypto.SetWrappedMetadataKey(encryptionEngine, envelope)
    } else {
        crypto.SetMetadataKey(encryptionEngine, metadataKey)
    }
    zeroBytes(metadataKey)
}
```

## 3. Security Analysis

### 3.1 Threat Model Update

| Threat | Before | After |
|--------|--------|-------|
| Attacker with S3 read access | Sees all encryption params, salt, KDF cost, manifest, original size | Sees only `encrypted: true` and opaque blob; no params visible |
| Attacker with S3 read + gateway password | Same | Attacker still needs metadata key (256-bit random or KMS) to decrypt blob |
| Attacker with S3 read + gateway host access | Same | Metadata key may be extracted from memory; this is the expected threat boundary |
| Attacker with KMS access only | N/A (no metadata encryption) | Can decrypt metadata if wrapping envelope is present; still needs S3 read |

### 3.2 Key Loss Scenario

Losing the metadata master key means the encrypted blob cannot be decrypted.
Without the blob, the following critical fields are unavailable:
- Per-object salt (needed to derive the data encryption key)
- Algorithm (needed to select AES-GCM or ChaCha20-Poly1305)
- Chunk manifest (needed for chunked decryption)
- Original size and ETag

**Result:** Objects become **permanently undecryptable** even if the main
encryption password is known.

**Mitigation:**
- Store the metadata key file with the same backup procedures as the main
  password
- When KMS wrapping is used, the KMS provider handles key durability and HA

### 3.3 Metadata Size Impact

| Scenario | Before | After | Delta |
|----------|--------|-------|-------|
| Uncompacted plaintext | ~880 bytes | ~880 bytes (blob replaces individual keys) | ~0 bytes |
| Compacted plaintext | ~580 bytes | ~580 bytes (blob replaces individual keys) | ~0 bytes |
| With large manifest | triggers fallback | triggers fallback (blob included in size calc) | ~0 bytes |

The encrypted blob is approximately the same size as the plaintext metadata it
replaces because:
- JSON serialization of the subset is ~350–400 bytes
- AES-GCM ciphertext adds 12 bytes nonce + 16 bytes tag = 28 bytes overhead
- Base64 encoding adds ~33% overhead
- Total: ~530–590 bytes, comparable to compacted plaintext

## 7. Migration Path

No migration of existing objects is required. Old objects (without
`x-amz-meta-enc-metadata`) continue to use the plaintext path. New objects use
the encrypted path when the feature is enabled.

If an operator wants to migrate existing objects:
1. Enable metadata encryption in the gateway config
2. Use `s3eg-migrate` with a new filter (future work: `--filter=metadata-encryption`)
3. The migration tool would read each object, decrypt data, re-encrypt with
   metadata encryption enabled, and overwrite

This is **not** part of the initial implementation and can be deferred to a
future release.
