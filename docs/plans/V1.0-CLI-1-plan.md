## 1. Context & Current State

### 1.1 Existing binary landscape

The project ships three binaries today:

| Binary | Entry point | Purpose |
|--------|-------------|---------|
| `s3-encryption-gateway` | `cmd/server/main.go` | Gateway server daemon |
| `s3eg-migrate` | `cmd/migrate/main.go` | Offline re-encryption migration tool |
| `healthcheck` | `cmd/healthcheck/main.go` | Kubernetes liveness probe |

None of these binaries is intended for interactive developer use. A developer who wants to verify that the gateway encrypts or decrypts correctly today must either write a full AWS SDK script or use `awscli`/`s5cmd` pointed at the gateway, which requires the gateway to be running with a valid credential store.

### 1.2 Missing developer experience

There is no quick-path tool that:
- Generates a pre-signed URL signed with gateway credentials (useful for testing time-limited URL access).
- Uploads a small file or inline bytes through the gateway's own authentication layer without requiring a full SDK setup.
- Downloads and prints an object to stdout for quick inspection.
- Inspects object metadata/headers without downloading the body.

The gateway's `internal/api/auth.go` handles SigV4 validation at `internal/api/auth.go:54` and presigned URL format at `internal/api/auth.go:65`. The `internal/s3` package exposes a `Client` interface (`internal/s3/client.go:35`) with `PutObject`, `GetObject`, `HeadObject` operations ‚Äî all suitable for direct CLI use via `aws-sdk-go-v2`.

### 1.3 Available dependencies

`go.mod` already lists:
- `github.com/aws/aws-sdk-go-v2/service/s3 v1.102.0` ‚Äî includes `s3.NewPresignClient` for presigned URL generation.
- `github.com/aws/aws-sdk-go-v2/config v1.32.18` ‚Äî credential loading.
- `github.com/aws/aws-sdk-go-v2/credentials v1.19.17` ‚Äî static credential provider.
- `github.com/spf13/cobra v1.10.2` (indirect; pulled in by `go-gremlins`) ‚Äî available for command dispatch without adding a new direct dependency.
- `github.com/sirupsen/logrus v1.9.4` ‚Äî structured output used by all other binaries.

`cobra` is already present as an indirect dependency; promoting it to a direct `require` entry for `cmd/cli` is the only `go.mod` change required.

### 1.4 Conventions observed

From `cmd/migrate/main.go`:
- Flag-based argument parsing using `flag.FlagSet` per sub-command.
- `--config` flag defaulting to `gateway.yaml`; gateway config loaded via `config.LoadConfig`.
- `--log-level` and `--output` (text/json) flags on every sub-command.
- `log/slog` for structured output with a shared `newLogger(level, format)` helper.
- `os.Exit(1)` on fatal errors.
- `context.Context` with `signal.NotifyContext` for cancellation.

The CLI will follow these same conventions and add Cobra's sub-command dispatch for discoverable help text.

### 1.5 Why this binary is net-new, not a flag on the server

The server binary (`cmd/server/main.go`) initialises the full gateway stack (encryption engine, credential store, metrics, tracing, Valkey) ‚Äî an operation that takes several seconds even for local MinIO. The CLI must start in milliseconds. A separate binary avoids pulling in that startup complexity entirely.

## 2. Design Goals & Non-Goals

### Goals

1. **Presign command** ‚Äî generate a SigV4 presigned GET or PUT URL for a given object, signed with gateway credentials, with a configurable expiry duration; print the URL to stdout.
2. **Put command** ‚Äî upload a file (or `-` for stdin) to a bucket/key through the gateway, supporting arbitrary metadata key=value pairs via repeated flags.
3. **Get command** ‚Äî download an object from the gateway to a local file or stdout; print HTTP response metadata to stderr in verbose mode.
4. **Head command** ‚Äî fetch object metadata (HTTP headers / S3 metadata) and print as key=value pairs; exit non-zero if the object does not exist.
5. **Credential parity** ‚Äî all commands accept `--access-key`, `--secret-key`, `--endpoint` flags (overriding config file values) so the tool works without a `gateway.yaml` on disk.
6. **Config-file integration** ‚Äî when a `gateway.yaml` is present, the CLI reads `backend.endpoint`, `backend.access_key`, `backend.secret_key`, and `backend.use_ssl` to pre-populate flags, preserving the single-source-of-truth property already used by `s3eg-migrate`.
7. **Zero new runtime dependencies** ‚Äî use only packages already in `go.mod`; only `github.com/spf13/cobra` needs promotion from indirect to direct.

### Non-Goals

1. **Encryption in the CLI** ‚Äî the CLI targets the gateway over HTTP/HTTPS; it does not encrypt/decrypt locally. Encryption is the gateway's responsibility. Deferred: if a local-encryption mode is ever needed it belongs in a follow-up issue.
2. **Bucket management** (create, delete, list buckets) ‚Äî out of scope for a developer-testing tool; tracked separately if needed.
3. **Multipart uploads** ‚Äî the `put` command uses single-part `PutObject`; large-file multipart support is deferred to a follow-up.
4. **Shell completion scripts** ‚Äî Cobra can generate them; deferred to a follow-up polish issue.
5. **Windows binary in CI** ‚Äî multi-arch build covers linux/amd64, linux/arm64, darwin/arm64 (matching the existing pattern); Windows is not in scope for v1.0.

## 3. Target Interface / Semantics / Behaviour

### 3.1 Binary name and install path

The binary is named `s3eg-cli` (matching the `s3eg-migrate` naming convention).

Makefile target: `cli` ‚Üí builds `bin/s3eg-cli-$(VERSION)`.

### 3.2 Top-level command structure

```
s3eg-cli [global flags] <command> [command flags] [args]

Global flags:
  --config string       path to gateway config file (default "gateway.yaml")
  --endpoint string     gateway endpoint URL (overrides config)
  --access-key string   gateway access key (overrides config)
  --secret-key string   gateway secret key (overrides config)
  --region string       AWS region for SigV4 (default "us-east-1")
  --log-level string    log level: debug, info, warn, error (default "info")
  --output string       output format: text, json (default "text")

Commands:
  presign   Generate a pre-signed URL for an S3 object
  put       Upload a file to the gateway
  get       Download an object from the gateway
  head      Fetch object metadata from the gateway
  version   Print the CLI version and exit
  help      Help about any command
```

### 3.3 `presign` command

```
s3eg-cli presign <bucket> <key> [flags]

Flags:
  --method string     HTTP method to sign: GET | PUT (default "GET")
  --expires duration  URL validity duration (default 15m; max 7 days)

Output (text):  the presigned URL printed to stdout, newline-terminated.
Output (json):  {"url":"https://...", "expires_at":"RFC3339", "method":"GET"}

Exit codes:
  0  URL generated and printed.
  1  Configuration error (missing credentials, invalid flags).
```

**Semantics:** Uses `s3.NewPresignClient` from `github.com/aws/aws-sdk-go-v2/service/s3` with `aws/credentials.NewStaticCredentialsProvider` populated from the resolved gateway credentials. The presigned URL is signed for the **gateway** endpoint (not the backend), so it can be used by a third party to fetch an object directly from the gateway within the expiry window. The URL format is standard SigV4 presigned (compatible with `curl`).

### 3.4 `put` command

```
s3eg-cli put <bucket> <key> [flags]

Positional:
  bucket  destination bucket name
  key     destination object key

Flags:
  --file string             path to local file; use "-" for stdin (required)
  --content-type string     MIME type (default "application/octet-stream")
  --metadata strings        metadata key=value pairs; may be repeated
                            e.g. --metadata author=alice --metadata env=test
  --tags string             URL-encoded tag set e.g. "env=test&team=devex"

Output (text):  "uploaded <bucket>/<key> (<N> bytes)\n" to stdout.
Output (json):  {"bucket":"...","key":"...","bytes":N}

Exit codes:
  0  Object uploaded successfully.
  1  Configuration or file error.
  2  Gateway returned HTTP error (status code printed to stderr).
```

**Semantics:** Opens the specified file (or `os.Stdin` when `--file=-`), wraps it in an `io.Reader`, and issues an authenticated `PutObject` against the gateway endpoint via `aws-sdk-go-v2`. Content-Length is set from `os.Stat` when the source is a file; omitted (chunked transfer) for stdin. The file is uploaded via the HTTP interface ‚Äî the gateway's encryption layer is applied transparently.

### 3.5 `get` command

```
s3eg-cli get <bucket> <key> [flags]

Flags:
  --output string       destination file path; use "-" for stdout (default "-")
  --version-id string   specific object version ID (optional)
  --verbose             print response headers to stderr before body

Output:  object body written to --output destination.

Exit codes:
  0  Object downloaded successfully.
  1  Configuration error.
  2  Object not found (NoSuchKey) or gateway error.
```

**Semantics:** Issues a `GetObject` request, streams the response body to the output destination. If `--verbose` is set, HTTP response headers (x-amz-meta-*, content-type, etag) are emitted to stderr as `key: value` lines before the body begins.

### 3.6 `head` command

```
s3eg-cli head <bucket> <key> [flags]

Flags:
  --version-id string   specific object version ID (optional)

Output (text):  one "key: value" line per metadata entry, sorted by key, to stdout.
Output (json):  {"metadata":{"key":"value",...},"etag":"...","content_length":N}

Exit codes:
  0  Object exists; metadata printed.
  1  Configuration error.
  2  Object not found (NoSuchKey).
  3  Other gateway error.
```

### 3.7 Configuration resolution precedence

For each config value the resolution order (highest priority first) is:

1. Explicit CLI flag (e.g. `--endpoint`)
2. Environment variable (e.g. `BACKEND_ENDPOINT`)
3. `gateway.yaml` field (loaded via `config.LoadConfig` if the file exists)
4. Compiled-in default

When the gateway config file is absent and no flags are provided for credentials, the CLI exits with an error and a clear message: `"credentials required: provide --access-key/--secret-key or set in gateway.yaml"`.

### 3.8 Error table

| Condition | Exit code | stderr message |
|-----------|-----------|---------------|
| No credentials | 1 | `credentials required: provide --access-key/--secret-key` |
| Invalid endpoint URL | 1 | `invalid endpoint: <url>: <detail>` |
| File not found (put) | 1 | `file not found: <path>` |
| Object not found (get/head) | 2 | `NoSuchKey: <bucket>/<key>` |
| Gateway HTTP error | 2 | `gateway error <STATUS>: <body excerpt>` |
| Unknown flag / bad args | 1 | (Cobra default usage print) |

## 4. Work Breakdown

### Phase A ‚Äî Project scaffolding (`cmd/cli`)

- **A1.** `cmd/cli/main.go` ‚Äî **new** ‚Äî entry point; calls `cli.Execute()` and exits with the returned code. Embeds `version`/`commit` ldflags matching the server binary pattern.
- **A2.** `cmd/cli/root.go` ‚Äî **new** ‚Äî Cobra root command; declares global persistent flags (`--config`, `--endpoint`, `--access-key`, `--secret-key`, `--region`, `--log-level`, `--output`); `PersistentPreRunE` resolves config from file+flags+env and stores a `*cliConfig` in the Cobra context.

  ```go
  // cliConfig is the resolved CLI configuration passed via cobra.Command context.
  type cliConfig struct {
      Endpoint  string
      AccessKey string
      SecretKey string
      Region    string
      LogLevel  string
      Output    string // "text" | "json"
  }

  // resolveConfig merges config file, flags, and env vars.
  // Precedence: flag > env > config file > default.
  func resolveConfig(cmd *cobra.Command, cfgPath string) (*cliConfig, error)
  ```

- **A3.** `cmd/cli/s3client.go` ‚Äî **new** ‚Äî `buildS3Client(cfg *cliConfig) (*s3.Client, error)` constructs an `aws-sdk-go-v2` S3 client with `credentials.NewStaticCredentialsProvider(cfg.AccessKey, cfg.SecretKey, "")`, custom endpoint resolver, and `UsePathStyle: true` (matching MinIO/gateway convention). Returns the typed SDK client (not the `internal/s3.Client` interface ‚Äî the CLI talks HTTP directly to the gateway).

  ```go
  func buildS3Client(cfg *cliConfig) (*s3.Client, error) {
      creds := credentials.NewStaticCredentialsProvider(cfg.AccessKey, cfg.SecretKey, "")
      awsCfg, err := awsconfig.LoadDefaultConfig(context.Background(),
          awsconfig.WithCredentialsProvider(creds),
          awsconfig.WithRegion(cfg.Region),
      )
      if err != nil {
          return nil, fmt.Errorf("build aws config: %w", err)
      }
      return s3.NewFromConfig(awsCfg, func(o *s3.Options) {
          o.BaseEndpoint = aws.String(cfg.Endpoint)
          o.UsePathStyle = true
      }), nil
  }
  ```

### Phase B ‚Äî `presign` command

- **B1.** `cmd/cli/cmd_presign.go` ‚Äî **new** ‚Äî Cobra command `presign`; flags: `--method` (default "GET"), `--expires` (default 15m). Validates that `--method` is GET or PUT. Constructs `s3.NewPresignClient(s3Client)`. Calls `PresignGetObject` or `PresignPutObject` with `s3.WithPresignExpires(cfg.Expires)`. Prints result according to `--output` mode.
- **B2.** Validate expiry bounds: `--expires` must be in `[1s, 168h]` (7 days); reject values outside this range with exit code 1 and a clear error message.

### Phase C ‚Äî `put` command

- **C1.** `cmd/cli/cmd_put.go` ‚Äî **new** ‚Äî Cobra command `put`; flags: `--file` (required), `--content-type`, `--metadata` (StringArray), `--tags`. Opens file or `os.Stdin` when `--file=-`. Calls `s3Client.PutObject`. On success prints upload confirmation. On error prints gateway status and exits 2.
- **C2.** Metadata parsing: each `--metadata key=value` entry is split on the first `=`; entries without `=` are rejected with exit code 1. Metadata map is passed as `Metadata` in the SDK `PutObjectInput`.

### Phase D ‚Äî `get` command

- **D1.** `cmd/cli/cmd_get.go` ‚Äî **new** ‚Äî Cobra command `get`; flags: `--output` (default "-"), `--version-id`, `--verbose`. Calls `s3Client.GetObject`. Streams `Output.Body` to the destination file (or `os.Stdout`). In verbose mode emits response metadata to `os.Stderr` before the body. Returns exit 2 on NoSuchKey, exit 2 on other errors.

### Phase E ‚Äî `head` command

- **E1.** `cmd/cli/cmd_head.go` ‚Äî **new** ‚Äî Cobra command `head`; flags: `--version-id`. Calls `s3Client.HeadObject`. Marshals metadata map to key=value (text) or JSON object. Exits 2 on NoSuchKey, 3 on other errors.

### Phase F ‚Äî Makefile & build integration

- **F1.** `Makefile` ‚Äî **edit** ‚Äî add `cli` target:

  ```makefile
  cli:
  	@echo "Building s3eg-cli..."
  	@CGO_ENABLED=0 go build -ldflags="-w -s -X main.version=$(VERSION) -X main.commit=$(COMMIT)" \
  		-o bin/s3eg-cli-$(VERSION) ./cmd/cli
  ```

- **F2.** `Makefile` ‚Äî **edit** ‚Äî add `cli-multiarch` target matching the `migrate-multiarch` pattern:

  ```makefile
  cli-multiarch:
  	@echo "Building s3eg-cli for multiple architectures..."
  	@GOOS=linux  GOARCH=amd64  CGO_ENABLED=0 go build -ldflags="-w -s" -o bin/s3eg-cli-linux-amd64  ./cmd/cli
  	@GOOS=linux  GOARCH=arm64  CGO_ENABLED=0 go build -ldflags="-w -s" -o bin/s3eg-cli-linux-arm64  ./cmd/cli
  	@GOOS=darwin GOARCH=arm64  CGO_ENABLED=0 go build -ldflags="-w -s" -o bin/s3eg-cli-darwin-arm64 ./cmd/cli
  ```

- **F3.** `Makefile` ‚Äî **edit** ‚Äî add `cli` and `cli-multiarch` to the `help` target's list of commands; add both to the `.PHONY` line.

### Phase G ‚Äî go.mod promotion

## 7. Risks & Mitigations

| Risk | Mitigation |
|---|---|
| `cobra` version mismatch between indirect and direct entry | Pin the exact version already in `go.sum` (`v1.10.2`); run `go mod tidy` and verify `go.sum` is not modified. |
| `--secret-key` appearing in shell history | Document in `docs/CLI.md` that users should prefer `BACKEND_SECRET_KEY` env var or a gateway.yaml with restrictive file permissions; add a `--secret-key-env` flag that reads from a named env var (Phase A3 extension). |
| `PutObject` with unknown content-length for stdin causes a `413` from some gateways | Use `aws.Int64(-1)` to trigger chunked transfer encoding when `--file=-`; document this behaviour in CLI.md. |
| Presigned URL signed against the gateway credential instead of the backend | Explicitly confirmed as correct: the URL is for gateway consumption, not backend direct access. Documented in ¬ß3.3. |
| `config.LoadConfig` returns error when `gateway.yaml` is absent | Wrap the load in a try-load pattern: if the file is absent (os.IsNotExist) skip silently; only fail if the file exists but is malformed. |
| Integration tests require Docker | Guard all integration tests with `//go:build integration`; exclude from default `make test`; include in a separate `make test-cli-integration` target. |
