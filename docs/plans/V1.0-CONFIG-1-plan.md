# V1.0-CONFIG-1 — Per-Bucket Encryption Bypass + Flat ENV Policy Configuration — Implementation Plan

Status: Draft
Owner: Config / DX / Crypto
Priority: P2
Labels: `milestone:v1.0`, `area:config`, `area:crypto`, `area:dx`

> The GitHub issue (#196) asked "can I set a policy via ENV to skip encryption
> for a bucket?" Two separate gaps need to be closed: (1) **per-bucket
> encryption bypass does not exist at all** — the gateway encrypts every object
> unconditionally; (2) **the only way to supply a policy is a mounted file**
> via the `POLICIES` env var, which is impractical in Kubernetes/Docker
> environments where operators prefer to express configuration entirely through
> environment variables. This plan addresses both.

---

## 1. Context & Current State

### 1.1 The encrypt-everything invariant

Every write path in the gateway calls `engine.Encrypt` unconditionally.
`getEncryptionEngine` (`internal/api/handlers.go:696–772`) is the central
dispatch: it returns the global engine when no policy matches a bucket, and
builds a per-policy engine (different key or algorithm) when a policy matches.
There is no branch that returns a no-op engine. Every path produces real AEAD
ciphertext.

The only existing partial bypass is `encrypt_multipart_uploads: false`
(`internal/config/policy.go:29–32`) which opts a bucket out of the Valkey/DEK
MPU path. Single-PUT objects to that same bucket are still encrypted.
`EncryptMultipartUploads` is the only `PolicyConfig` field that can prevent
encryption — and only for multipart uploads.

### 1.2 The policy-as-file-only constraint

`Config.PolicyFiles` (`internal/config/config.go:40`) holds file paths.
`loadFromEnv` (`internal/config/config.go:1424–1426`) parses `POLICIES` as a
comma-separated list of **file paths** only. Every other config section has
flat env vars (e.g. `BACKEND_ENDPOINT`, `ENCRYPTION_PASSWORD`). The policy
system uniquely requires a mounted file, which is a friction point for
container deployments.

The existing `GW_CRED_N_*` pattern (`config.go:1445–1468`) shows the
established convention for expressing a list of structured objects via indexed
env vars. `GW_CRED_0_ACCESS_KEY` / `GW_CRED_0_SECRET_KEY` iterate until both
are absent at index N. This same pattern applied to policies yields
`GW_POLICY_N_ID` as the sentinel.

### 1.3 `PolicyConfig` fields today

```
internal/config/policy.go:13–32
```

| Field | Type | Yaml tag | Env coverage today |
|---|---|---|---|
| `ID` | `string` | `id` | none |
| `Buckets` | `[]string` | `buckets` | none |
| `Encryption` | `*EncryptionConfig` | `encryption` | none (complex struct) |
| `RateLimit` | `*RateLimitConfig` | `rate_limit` | none |
| `RequireEncryption` | `bool` | `require_encryption` | none |
| `DisallowLockBypass` | `bool` | `disallow_lock_bypass` | none |
| `EncryptMultipartUploads` | `*bool` | `encrypt_multipart_uploads` | none |

No field has an env var equivalent. `DisableEncryption` does not exist yet.

### 1.4 The `EncryptionEngine` interface

`internal/crypto/engine.go:74–93` — five methods must be satisfied by any
passthrough implementation:

```go
Encrypt(ctx, reader, metadata) (io.Reader, map[string]string, error)
Decrypt(ctx, reader, metadata) (io.Reader, map[string]string, error)
DecryptRange(ctx, reader, metadata, start, end) (io.Reader, map[string]string, error)
IsEncrypted(metadata) bool
PreferredAlgorithm() string
```

The existing `engine.Decrypt` at `engine.go:636–638` already has a read-side
passthrough for unencrypted objects (`if !e.IsEncrypted(metadata) { return reader, metadata, nil }`).
This means objects stored without encryption headers will GET correctly through
the standard engine without changes. The problem is entirely on the **write
path**.

### 1.5 What `ApplyToConfig` currently does

`internal/config/policy.go:115–151` merges `policy.Encryption` fields
(password, algorithm, key manager) onto a copy of the base config. It is an
override mechanism, not a bypass: you cannot set `Encryption = nil` to disable
encryption; the merge is always additive.

## 2. Design Goals & Non-Goals

### Goals

1. **Explicit encryption opt-out** — `disable_encryption: true` in a
   `PolicyConfig` makes the gateway store and serve objects as plain bytes for
   matching buckets. No silent passthrough; the field must be declared.
2. **Fail-safe on mixed content** — a bypass bucket that already contains
   encrypted objects (written before the policy was applied) must return a
   clear HTTP error on GET, not silently serve ciphertext as data.
3. **Conflict detection** — `disable_encryption: true` + `require_encryption: true`
   in the same policy is rejected as a fatal startup error.
4. **Full ENV coverage via `GW_POLICY_N_*`** — every field of `PolicyConfig`
   (including the new `disable_encryption`) is expressible as a flat,
   indexed env var. No file mount needed for any policy use-case.
5. **`GW_POLICY_N_*` mirrors `GW_CRED_N_*`** — same sentinel logic (stop when
   `GW_POLICY_N_ID` is absent), same `loadFromEnv` location, same hot-reload
   semantics.
6. **Consistent validation** — env-sourced policies pass the same validation as
   file-sourced policies (ID required, ≥1 bucket pattern required, conflict
   check).

### Non-Goals

1. **Transparent re-decryption of pre-existing encrypted objects** in a bypass
   bucket. That is a migration concern (V1.0-MAINT-1).
2. **Per-prefix bypass** — bypass is per-bucket only.
3. **`EncryptionConfig` sub-fields via env** — `GW_POLICY_N_ENCRYPTION_PASSWORD`
   and `GW_POLICY_N_ENCRYPTION_ALGORITHM` cover the two most common per-bucket
   encryption overrides. Full `KeyManagerConfig` sub-fields are out of scope
   for this plan; operators needing per-bucket KMS config still use a policy
   file.

## 3. Target Interface / Semantics / Behaviour

### 3.1 New `PolicyConfig` field

```go
// DisableEncryption, when true, causes the gateway to store and serve objects
// in matching buckets as plain bytes without AEAD encryption.
//
// Invariants:
//   - Mutually exclusive with RequireEncryption: true. LoadPolicies (and
//     LoadPoliciesFromEnv) returns an error if both are set.
//   - A GET on an object that was stored before this flag was set — i.e. one
//     carrying x-amz-meta-encrypted: "true" — returns HTTP 409
//     EncryptionConfigurationMismatch.
//   - Default false; all existing behaviour is preserved.
//   - Also implies EncryptMultipartUploads = false; operators need not set both.
DisableEncryption bool `yaml:"disable_encryption,omitempty"`
```

### 3.2 `PassthroughEngine` (`internal/crypto/passthrough.go`)

```go
// PassthroughEngine is a no-op EncryptionEngine for buckets with
// disable_encryption: true.
//
// Invariants:
//   - Encrypt:      returns reader and metadata unchanged; no encryption headers.
//   - Decrypt:      if !IsEncryptedMetadata(meta), returns reader unchanged.
//                   if IsEncryptedMetadata(meta), returns ErrEncryptedObjectInBypassBucket.
//   - DecryptRange: same semantics as Decrypt.
//   - IsEncrypted:  always returns false (bypass engine never marks objects).
//   - PreferredAlgorithm: returns "none".
//   - Thread-safe: zero-value struct, no state.
type PassthroughEngine struct{}

var ErrEncryptedObjectInBypassBucket = errors.New(
    "object was stored encrypted; bucket now has disable_encryption — " +
    "use the migration tool to re-store as plaintext")
```

**Why a separate type rather than a nil check in each handler?**
`PassthroughEngine` as a type satisfying `EncryptionEngine` means every
handler (`handlePutObject`, `handleGetObject`, `handleUploadPart`,
`handleCopyObject`) calls `engine.Encrypt` / `engine.Decrypt` with no
conditional branching at the call site. The bypass is fully encapsulated
inside `getEncryptionEngine`.

### 3.3 `getEncryptionEngine` change

One early-return inserted at `internal/api/handlers.go` immediately after
`policy := h.policyManager.GetPolicyForBucket(bucket)` returns non-nil
(currently line ~702):

```go
if policy.DisableEncryption {
    return crypto.PassthroughEngine{}, nil   // stateless; never cached
}
```

`PassthroughEngine` is stateless so it is not stored in `h.engineCache`.

### 3.4 HTTP error for encrypted object in bypass bucket

`ErrEncryptedObjectInBypassBucket` from `PassthroughEngine.Decrypt` /
`DecryptRange` must be translated to HTTP 409 with S3 error code
`EncryptionConfigurationMismatch` in the handler error paths:

```
HTTP/1.1 409 Conflict
<?xml version="1.0" encoding="UTF-8"?>
<Error>
  <Code>EncryptionConfigurationMismatch</Code>
  <Message>The object was encrypted when stored but the bucket policy now
  disables encryption. Use the migration tool to convert the object.</Message>
</Error>
```

This mirrors AWS S3's own behaviour when a GET violates the bucket's SSE
configuration [1].

### 3.5 `BucketEncryptsMultipart` change

When `DisableEncryption` is true, `BucketEncryptsMultipart` must return
`false` — the operator should not need to set two flags to disable all
encryption for a bucket:

```go
// internal/config/policy.go — BucketEncryptsMultipart (line ~156)
if policy.DisableEncryption {
    return false
}
```

### 3.6 `GW_POLICY_N_*` env var contract

Modelled directly after `GW_CRED_N_*` (`config.go:1445–1468`).

Iteration starts at N=0 and stops when `GW_POLICY_N_ID` is absent.

| Variable | Type | Notes |
|---|---|---|
| `GW_POLICY_N_ID` | string | Required; sentinel for iteration stop |
| `GW_POLICY_N_BUCKETS` | comma-separated strings | Required; ≥1 glob pattern |
| `GW_POLICY_N_DISABLE_ENCRYPTION` | `true`\|`false` | Default false |
| `GW_POLICY_N_REQUIRE_ENCRYPTION` | `true`\|`false` | Default false |
| `GW_POLICY_N_DISALLOW_LOCK_BYPASS` | `true`\|`false` | Default false |
| `GW_POLICY_N_ENCRYPT_MULTIPART_UPLOADS` | `true`\|`false` | Unset if absent (nil pointer semantics preserved) |
| `GW_POLICY_N_ENCRYPTION_PASSWORD` | string | Sets `Encryption.Password` |
| `GW_POLICY_N_ENCRYPTION_ALGORITHM` | string | Sets `Encryption.PreferredAlgorithm` |
| `GW_POLICY_N_RATE_LIMIT_ENABLED` | `true`\|`false` | Activates rate-limit override |
| `GW_POLICY_N_RATE_LIMIT_REQUESTS` | int | Sets `RateLimit.Limit` |
| `GW_POLICY_N_RATE_LIMIT_WINDOW` | duration string (e.g. `1m`) | Sets `RateLimit.Window` |

**Example — Restic bypass, no file needed:**
```bash
GW_POLICY_0_ID=restic-bypass
GW_POLICY_0_BUCKETS=restic-*
GW_POLICY_0_DISABLE_ENCRYPTION=true
```

**Example — per-bucket key override:**
```bash
GW_POLICY_0_ID=finance-bucket
GW_POLICY_0_BUCKETS=finance
GW_POLICY_0_ENCRYPTION_PASSWORD=s3cr3t-finance-key
GW_POLICY_0_ENCRYPTION_ALGORITHM=AES-256-GCM
```

**Merge semantics:** Env-sourced policies are loaded after file-sourced
policies. Both sets are merged additively in the same `pm.policies` slice.
First-match-wins semantics are unchanged.

### 3.7 `PolicyManager.LoadPoliciesFromEnv()`

```go
// LoadPoliciesFromEnv reads GW_POLICY_N_* indexed env vars and appends the
// resulting policies to pm.policies. Iteration stops at the first N where
// GW_POLICY_N_ID is absent. Validation is identical to LoadPolicies.
//
// Invariants:
//   - Thread-safe (acquires pm.mu.Lock).
//   - Appends to existing policies; call Reset() before a full reload.
//   - Returns the first validation error encountered.
func (pm *PolicyManager) LoadPoliciesFromEnv() error
```

### 3.8 `PolicyManager.Reset()`

```go
// Reset clears all loaded policies. Must be called before a full config
// reload to prevent policy accumulation across SIGHUP cycles.
func (pm *PolicyManager) Reset()
```

- **A2.** `internal/crypto/passthrough.go` — **new file**

  ```go
  package crypto

  import (
      "context"
      "errors"
      "io"
  )

  var ErrEncryptedObjectInBypassBucket = errors.New(
      "object was stored encrypted; bucket now has disable_encryption — " +
      "use the migration tool to re-store as plaintext")

  // Compile-time interface check.
  var _ EncryptionEngine = PassthroughEngine{}

  type PassthroughEngine struct{}

  func (PassthroughEngine) Encrypt(_ context.Context, r io.Reader, meta map[string]string) (io.Reader, map[string]string, error) {
      return r, meta, nil
  }

  func (PassthroughEngine) Decrypt(_ context.Context, r io.Reader, meta map[string]string) (io.Reader, map[string]string, error) {
      if IsEncryptedMetadata(meta) {
          return nil, nil, ErrEncryptedObjectInBypassBucket
      }
      return r, meta, nil
  }

  func (PassthroughEngine) DecryptRange(_ context.Context, r io.Reader, meta map[string]string, _, _ int64) (io.Reader, map[string]string, error) {
      if IsEncryptedMetadata(meta) {
          return nil, nil, ErrEncryptedObjectInBypassBucket
      }
      return r, meta, nil
  }

  func (PassthroughEngine) IsEncrypted(_ map[string]string) bool { return false }
  func (PassthroughEngine) PreferredAlgorithm() string           { return "none" }
  ```

  ```go
  func (pm *PolicyManager) LoadPoliciesFromEnv() error {
      pm.mu.Lock()
      defer pm.mu.Unlock()

      for i := 0; ; i++ {
          id := os.Getenv(fmt.Sprintf("GW_POLICY_%d_ID", i))
          if id == "" {
              break
          }
          bucketsRaw := os.Getenv(fmt.Sprintf("GW_POLICY_%d_BUCKETS", i))
          if bucketsRaw == "" {
              return fmt.Errorf("GW_POLICY_%d_BUCKETS must specify at least one pattern", i)
          }
          buckets := splitTrimmed(bucketsRaw, ",")

          policy := &PolicyConfig{ID: id, Buckets: buckets}

          if v := os.Getenv(fmt.Sprintf("GW_POLICY_%d_DISABLE_ENCRYPTION", i)); v != "" {
              policy.DisableEncryption = v == "true" || v == "1"
          }
          if v := os.Getenv(fmt.Sprintf("GW_POLICY_%d_REQUIRE_ENCRYPTION", i)); v != "" {
              policy.RequireEncryption = v == "true" || v == "1"
          }
          if v := os.Getenv(fmt.Sprintf("GW_POLICY_%d_DISALLOW_LOCK_BYPASS", i)); v != "" {
              policy.DisallowLockBypass = v == "true" || v == "1"
          }
          if v := os.Getenv(fmt.Sprintf("GW_POLICY_%d_ENCRYPT_MULTIPART_UPLOADS", i)); v != "" {
              b := v == "true" || v == "1"
              policy.EncryptMultipartUploads = &b
          }

          // Encryption sub-fields
          encPwd := os.Getenv(fmt.Sprintf("GW_POLICY_%d_ENCRYPTION_PASSWORD", i))
          encAlg := os.Getenv(fmt.Sprintf("GW_POLICY_%d_ENCRYPTION_ALGORITHM", i))
          if encPwd != "" || encAlg != "" {
              enc := &EncryptionConfig{}
              enc.Password = encPwd
              enc.PreferredAlgorithm = encAlg
              policy.Encryption = enc
          }

          // Rate-limit sub-fields
          rlEnabled := os.Getenv(fmt.Sprintf("GW_POLICY_%d_RATE_LIMIT_ENABLED", i))
          rlReq := os.Getenv(fmt.Sprintf("GW_POLICY_%d_RATE_LIMIT_REQUESTS", i))
          rlWin := os.Getenv(fmt.Sprintf("GW_POLICY_%d_RATE_LIMIT_WINDOW", i))
          if rlEnabled != "" || rlReq != "" || rlWin != "" {
              rl := &RateLimitConfig{}
              rl.Enabled = rlEnabled == "true" || rlEnabled == "1"
              if n, err := strconv.Atoi(rlReq); err == nil && n > 0 {
                  rl.Limit = n
              }
              if d, err := time.ParseDuration(rlWin); err == nil {
                  rl.Window = d
              }
              policy.RateLimit = rl
          }

          if policy.DisableEncryption && policy.RequireEncryption {
              return fmt.Errorf("GW_POLICY_%d (%q): disable_encryption and require_encryption are mutually exclusive", i, id)
          }

          pm.policies = append(pm.policies, policy)
      }
      return nil
  }
  ```

  `splitTrimmed` is a package-private helper (or inline using
  `strings.Split` + range trim) — consistent with how `loadFromEnv` handles
  comma-separated values at `config.go:980–985`.

  1. Always call `policyManager.Reset()` before reloading.
  2. Call both `LoadPolicies` and `LoadPoliciesFromEnv`.
  3. Detect env-policy changes via a new `envPoliciesChanged(old, new *Config) bool`
     helper (see note below).

  ```go
  // Reload policies whenever file list changes OR any GW_POLICY_N_* env var
  // changes (env changes are detected by re-reading live env at reload time,
  // not by comparing old/new config structs).
  policyFileChanged := !stringSlicesEqual(oldCfg.PolicyFiles, newCfg.PolicyFiles)
  if policyFileChanged || envPoliciesChanged() {
      a.policyManager.Reset()
      if len(newCfg.PolicyFiles) > 0 {
          if err := a.policyManager.LoadPolicies(newCfg.PolicyFiles); err != nil { ... }
      }
      if err := a.policyManager.LoadPoliciesFromEnv(); err != nil { ... }
      changes = append(changes, "policies reloaded")
  }
  ```

  `envPoliciesChanged()` simply checks whether `GW_POLICY_0_ID` is set (or
  compares the current env snapshot with what was loaded last time). A simple
  and correct approach: always re-run `LoadPoliciesFromEnv` on every SIGHUP
  regardless of whether env vars changed — the cost is negligible and avoids
  needing to snapshot env vars.

  **Simpler alternative (preferred):** always reload both sources on any
  SIGHUP (remove the `policyFileChanged` guard entirely for policy reloads).
  Policies are cheap to re-parse; always doing so is safer than missing an env
  change.

## 7. Risks & Mitigations

| Risk | Mitigation |
|---|---|
| Operator sets `disable_encryption: true` on wrong bucket; sensitive data stored plaintext | Field requires explicit declaration; a `WARN`-level log is emitted at startup for every loaded bypass policy listing the bucket patterns |
| Bypass bucket contains pre-existing encrypted objects; GET returns 409 unexpectedly | `ErrEncryptedObjectInBypassBucket` error message names the migration tool; HTTP 409 is the correct S3 error code — promoted to DoD |
| `Reset()` called during in-flight requests; request served with empty policy list | `GetPolicyForBucket` holds `pm.mu.RLock`; `Reset()` holds `pm.mu.Lock`; in-flight readers complete before reset takes effect — standard Go RWMutex guarantee |
| `LoadPoliciesFromEnv` accumulates across SIGHUP if `Reset()` is not called first | Hot-reload block always calls `Reset()` before both load functions — promoted to DoD |
| `GW_POLICY_N_ENCRYPTION_PASSWORD` leaks key material via process env | Existing security posture: `ENCRYPTION_PASSWORD` itself is an env var; same trade-off. Doc comment warns operators to prefer `GW_POLICY_N_*` + a Kubernetes Secret for sensitive values |
| `IsEncryptedMetadata` refactor in A1 changes behaviour of `engine.IsEncrypted` | The refactor is purely mechanical (extract body into named function; delegate); covered by existing `TestIsEncrypted_*` tests |

## 10. Follow-ups / Out-of-scope items surfaced during planning

- **Migration helper** — objects encrypted before `disable_encryption` was
  applied cannot be read in bypass mode (HTTP 409). A
  `migrate-to-plaintext` sub-command for `V1.0-MAINT-1` is the natural
  follow-up; out of scope here.
- **Full `KeyManagerConfig` via env** — `GW_POLICY_N_ENCRYPTION_PASSWORD` and
  `GW_POLICY_N_ENCRYPTION_ALGORITHM` cover the common cases. Per-bucket KMS
  provider wiring via env would require ~15 more variables; deferred until
  user demand is confirmed.
- **Audit events for bypass writes** — plaintext writes to bypass buckets
  should emit an audit event with `encryption_bypassed: true`. Deferred to
  V1.0-OBS-1.
- **Helm values schema** — `GW_POLICY_N_*` can be supplied via `extraEnv`
  today; an explicit `values.schema.json` entry is deferred.
